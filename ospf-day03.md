# Day 03：**LSDB 中的秘密**

昨天我们手动“卡死”了状态机，今天我们要钻进 OSPF 的灵魂——**LSDB（链路状态数据库）**。 很多新人觉得 LSA 难记，是因为他们只看书，不看数据库。

**今日重点：** **Type 1 (Router) 和 Type 2 (Network) LSA**。

- **比喻：** * **Type 1** 是“个人身份证”，每个路由器都会发，描述自己的接口和邻居。
  - **Type 2** 是“集体照”，由 DR 产生，描述这个广播网段里都有谁。

- **实操建议：** * 在你的模拟器里输入 `display ospf lsdb router 1.1.1.1`。
  - 重点看 **Link ID** 和 **Data** 字段。
  - **避坑点：** 很多人分不清 Link ID 到底填什么，今天我们要把这个总结出来。

**实验观察：LSDB 中的秘密**

- **Type-1 LSA (Router)**：在 R1 上执行 `display ospf lsdb router`。你会发现 Link ID 就是 R1 自己的 Router ID。它在向全网宣告：“我是 R1，我有这些接口。”
- **Type-2 LSA (Network)**：只有在广播网段（由 DR 产生）才能看到。它描述的是这个广播域里所有“兄弟”的名单。
- **思考**：为什么 P2P 链路上没有 Type-2 LSA？（因为点对点链路只有两个邻居，不需要 DR 来点名）。
