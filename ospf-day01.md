## 📝 OSPF 学习笔记 - Day 01：别再死记硬背了！像“入职面试”一样理解 OSPF 状态机

### 一句话人话解释

OSPF 建立邻居就像**新人入职**：从投简历（Down）到面试沟通（Init/2-Way），再到交换工号和熟悉业务（ExStart/Exchange/Loading），最后正式上岗干活（Full）。

------

### 核心干货：8 大状态全解析

| **状态 (State)** | **动作描述 (What happens?)** | **关键点**                             |
| ---------------- | ---------------------------- | -------------------------------------- |
| **Down**         | 初始状态                     | 还没收到任何 Hello 包。                |
| **Attempt**      | 尝试联系                     | 仅限 NBMA 网络，主动发包寻找邻居。     |
| **Init**         | 收到简历                     | 收到对方的 Hello，但对方还没发现我。   |
| **2-Way**        | 互相看对眼                   | 双方都发现对方，**DR/BDR 选举完成**。  |
| **ExStart**      | 谁是老大？                   | 决定 Master/Slave，同步序列号。        |
| **Exchange**     | 交换目录                     | 互相发送 DBD，告诉对方自己有的“书单”。 |
| **Loading**      | 查漏补缺                     | 请求具体的 LSA 条目（LSR/LSU/LSAck）。 |
| **Full**         | 正式合体                     | 数据库同步完成，路由计算开始。         |

------

### 避坑指南：为什么邻居总是卡住？

在现网维护中，如果看到邻居状态不正常，重点看这两个位置：

- **卡在 Init 状态？**
  - **原因：** 通常是“单相思”。我收到了对方的包，但对方没收到我的。
  - **排查：** 检查中间是否有防火墙拦截了 OSPF 报文（组播地址 224.0.0.5）。
- **卡在 ExStart/Exchange 状态？**
  - **原因 1：MTU 不匹配。** 如果双方接口 MTU 设置不一致且开启了 MTU 检查，会在这里“卡死”。
  - **原因 2：Router ID 冲突。** 两个路由器起了相同的名字，那是绝对过不下去的。

### 实验脚本：华为 CLI 常用命令

要在设备上查看邻居是否健康，记住这三行命令：


Bash

```

display ospf peer brief

display ospf peer
```
