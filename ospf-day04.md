- ### 📝 OSPF 学习笔记 - Day 04：特殊区域与路由汇总

  #### 1. 为什么要划分“特殊区域”？

  随着网络规模扩大，LSDB（链路状态数据库）会变得非常庞大。对于那些处于网络边缘、性能较弱的路由器（比如县公司接入网设备），如果接收全网的 LSA，CPU 和内存会不堪重负。

  - **解决方案**：通过减少不必要的 LSA 传播，保护边缘设备。

  #### 2. 四种特殊区域对比表

  | **区域类型**     | **允许的 LSA 类型** | **默认路由生成方式**     | **使用场景**                               |
  | ---------------- | ------------------- | ------------------------ | ------------------------------------------ |
  | **Stub** (末梢)  | 1, 2, 3             | 自动生成 3 类默认路由    | 只有一个出口，不接收外部路由。             |
  | **Totally Stub** | 1, 2                | 自动生成 3 类默认路由    | 极简模式，只保留区域内拓扑。               |
  | **NSSA**         | 1, 2, 3, 7          | 不自动生成（需手动配置） | 边缘区域连接了外部网络（如静默路由引入）。 |
  | **Totally NSSA** | 1, 2, 7             | 自动生成 3 类默认路由    | 既有外部连接，又要保持数据库精简。         |

  #### 3. 核心配置实验 (Huawei CLI)

  Bash

  ```
  # 配置 Stub 区域 (所有区域内路由器都要配)
  [Router] ospf 1
  [Router-ospf-1] area 1
  [Router-ospf-1-area-0.0.0.1] stub
  
  # 配置 Totally NSSA 区域 (在 ABR 上配置)
  [ABR] ospf 1
  [ABR-ospf-1] area 2
  [ABR-ospf-1-area-0.0.0.2] nssa no-summary
  ```

  #### 4. 路由汇总 (Route Summarization)

  为了进一步减少骨干区域（Area 0）的路由条目：

  - **ABR 汇总**：针对区域间的 3 类 LSA。
    - 命令：`abr-summary 192.168.0.0 255.255.0.0`
  - **ASBR 汇总**：针对引入的 5 类/7 类 LSA。
    - 命令：`asbr-summary 10.0.0.0 255.0.0.0`
